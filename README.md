## 🎯 프로젝트 목표

**생일 축하**를 <br />
_말 한 줄이 아니라, 잠깐 머물 수 있는 순간으로 만들고자 했어요._

- 만드는 사람도, 받는 사람도 <br />
  **부담 없이 즐길 수 있는 작은 이벤트가** 되었으면 했어요.

- 복잡한 가입이나 설명 없이도 <br />
  **바로 만들고, 바로 전할 수 있는** 흐름을 목표로 했어요.

## ✨ 주요 기능

> **생일 축하를**
> _말 한 줄이 아니라, 잠깐 머물 수 있는 순간으로 만들기 위해_
> 이런 기능들을 담았어요.

---

### ✍️ 제작자 경험 (Creator Experience)

#### 🎁 생일 선물을 준비하는 흐름

- 편지를 쓰는 대신,
  **하나의 생일 선물을 차근차근 준비하는 과정**으로 구성했어요.

- 단계마다 선택하고 완성해 가면서,
  메시지가 자연스럽게 만들어지도록 설계했어요.

#### 1. 생일 편지 작성

> - 파티의 마지막에 꺼내보게 될 편지라서,  
>   **조금 더 천천히, 정성을 담아 쓸 수 있도록** 했어요.
> - 단독으로 읽히는 글이 아니라,  
>   연출과 함께 감상되는 흐름을 고려했어요.

#### 2. 케이크 & 파티 연출

> - 케이크 위에 초를 직접 올리며,  
>   **축하하는 마음을 눈에 보이게 남길 수 있도록** 했어요.
> - 초의 개수를 제한해,  
>   상징성과 작은 재미를 함께 담았어요.

#### 3. 간편한 전달 & 보안

> - 완성되면 바로 링크 하나로 전달할 수 있어요.
> - 생일(MMDD)을 PIN으로 사용해,  
>   **초대받은 사람만 들어올 수 있는 파티**가 되도록 했어요.
> - 회원가입 없이도  
>   **부담 없이 시작하고, 가볍게 전할 수 있는 흐름**을 유지했어요.

---

### 🎉 수신자 경험 (Receiver Experience)

- 받는 사람에게도,
  이 축하가 **하나의 순간으로 남았으면 했어요.**

#### 🔐 초대 확인

> - 전달받은 링크로 들어오면,  
>   **초대받은 사람인지 한 번 더 확인하는 과정**이 있어요.
> - 생일(MMDD)을 PIN으로 입력해야  
>   파티가 열리도록 했어요.

#### 🎶 이벤트처럼 시작되는 축하

> - 인증이 끝나면,  
>   생일 축하 음악과 함께 케이크가 등장해요.
> - “초를 불어주세요”라는 안내와 함께,  
>   **이 파티의 가장 중요한 순간**이 시작돼요.
>   - 기본적으로는 마우스로 초를 끌 수 있고
>   - (확장 기능으로) 실제로 불어서 끄는 방식도 계획중이에요.

#### 🎈 파티를 감상하는 공간

> - 모든 초를 끄면,  
>   자연스럽게 파티 공간으로 이동해요.
> - 테이블 위에는 케이크와 편지가 놓여 있고,  
>   **신나는 음악과 함께 파티의 여운이 이어지는 공간**이 펼쳐져요.

#### 🖱️ 작은 상호작용들

> - 폭죽 버튼을 누르면,  
>   화면 가득 축하 컨페티가 터져요.
> - 케이크를 클릭하면 크게 감상할 수 있고,  
>   편지를 누르면 제작자가 남긴 메시지를 읽을 수 있어요.

#### 💾 추억으로 남는 카드

> - 이 파티의 연출은  
>   **처음 한 번만 경험하도록** 설계했어요.
> - 이후에는 링크에 다시 들어와도  
>   바로 파티 페이지로 이어져요.
> - 언제든 꺼내볼 수 있는,  
>   **기억처럼 남는 생일 카드**가 되었으면 했어요.

## � 프로젝트 구조

```
src/
├── components/          # 재사용 가능한 UI 컴포넌트
│   ├── Form/           # 폼 관련 컴포넌트 (날짜, 텍스트 입력)
│   ├── layout/         # 페이지 레이아웃 (프레임, 페이지)
│   ├── seo/            # SEO 유틸리티 및 설정
│   ├── shared/         # 공통 컴포넌트 (공유 버튼, 툴팁)
│   └── ui/             # 기본 UI 컴포넌트 (버튼, 입력, 헤더)
│
├── features/            # 기능 단위로 그룹화된 비즈니스 로직
│   ├── creator/         # 카드 제작 기능 (제작자 플로우)
│   ├── receiver/        # 카드 수신 기능 (수신자 플로우)
│   ├── constant/        # 기능 관련 상수
│   └── types/           # 기능 관련 타입 정의 (케이크, 초, 문자 등)

│   ├── useAudioToggle   # 오디오 토글 제어

│

│   ├── api/             # API 요청 로직 (제작자, 수신자)
│   ├── auth/            # 인증 관련 로직
│   └── share/           # 공유 기능 관련 로직
│
├── routes/              # TanStack Router 기반 라우트
│   ├── __root.tsx       # 루트 레이아웃
│   ├── creator/         # 제작자 페이지
│   ├── receiver/        # 수신자 페이지 (r/)
│   └── guides/          # 가이드 페이지
│
├── styles/              # 글로벌 스타일 및 디자인 토큰
│   ├── tokens/          # 색상, 폰트, 간격 등 토큰
│   └── utilities/       # 유틸리티 클래스
│
├── types/               # 글로벌 타입 정의
│   ├── cards.types.ts   # 카드 데이터 타입
│   ├── cardResult.types.ts  # 카드 결과 타입
│   └── ...              # 기타 글로벌 타입
│
├── utils/               # 유틸리티 함수
│   ├── formatPinBirth   # 생일 PIN 포매팅
```

### 주요 폴더 설명

| 폴더           | 설명                                                                                                                                              |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| **components** | 버튼, 입력 필드, 폼 등 재사용 가능한 UI 컴포넌트들을 관리합니다. 레이아웃, 공유 기능, SEO 관련 컴포넌트도 포함됩니다.                             |
| **features**   | 기능별로 구분된 비즈니스 로직입니다. 제작자 경험(creator)과 수신자 경험(receiver)으로 나뉘며, 각 기능의 컴포넌트, 데이터 처리, 유틸을 포함합니다. |

| **assets**     | 폰트, 오디오 파일 등 정적 자산을 관리합니다. (이미지는 `public/assets/`에 위치)                                                                   |

## 📖 서비스 플로우

> 이 서비스는
> **만드는 사람의 준비 과정**과
> **받는 사람의 경험 흐름**이 자연스럽게 이어지도록 설계되었어요.

```
[Creator]
  시작
    │
    ▼
  기본 정보 입력
    │
    ▼
  편지 작성
    │
    ▼
  케이크 선택
    │
    ▼
  케이크 제작 (초 배치)
    │
    ▼
  초대 카드 생성
    │
    ▼
────────── 링크 전달 ──────────▶

                              [Receiver]
                                │
                                ▼
                           초대 확인 (PIN)
                                │
                                ▼
                        케이크 등장 & 촛불 끄기
                                │
                                ▼
                           편지 감상
                                │
                                ▼
                           파티 페이지
```

---

### ✍️ 제작자 플로우 (Creator Flow)

> 생일 선물을 하나씩 준비해 나가는 과정이에요.

- **C1. 시작하기**
  랜딩 페이지에서 파티 준비를 시작해요.

- **C2. 기본 정보 입력**
  수신자의 이름과 생일(PIN)을 입력해요.

- **C3. 편지 작성**
  파티의 마지막에 전해질 생일 편지를 작성해요.

- **C4. 케이크 선택**
  파티에 어울리는 케이크를 고릅니다.

- **C5. 케이크 제작**
  케이크 위에 초를 직접 배치하며 축하의 의미를 담아요.

- **C6. 초대 카드 완성**
  모든 준비가 끝나면, 공유 가능한 초대 카드가 완성돼요.

---

### 🎉 수신자 플로우 (Receiver Flow)

> 초대를 받은 사람이
> 하나의 이벤트처럼 생일을 경험하는 흐름이에요.

- **R1. 초대 확인**
  링크 접속 후 생일(PIN)을 입력해 초대를 확인해요.

- **R2. 케이크 등장 & 촛불 끄기**
  음악과 함께 케이크가 등장하고,
  초를 끄는 핵심 이벤트가 진행돼요.

- **R3. 편지 감상**
  제작자가 준비한 생일 편지를 천천히 읽어요.

- **R4. 파티 페이지**
  축하의 여운이 남는 파티 공간으로 이어져요.
  이후 재접속 시에는 바로 이 페이지로 이동해요.
